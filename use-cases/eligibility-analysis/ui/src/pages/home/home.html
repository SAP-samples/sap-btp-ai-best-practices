Offer Analysis evaluates invoice eligibility against a fixed ruleset and classifies each invoice as Eligible only when all active rules pass. Rules considered
  • R1 - Days to Due Date (NDDT): Due date minus purchase date must be at least NDDT days.
  • R2 - Duplicate Invoice: Doc number + fiscal year + reference number must be unique in the batch.
  • R11 - Eligible Currency: Invoice currency must be in the configured eligible currency list.
  • R13 - Not Overdue: Due date must be strictly after the purchase date.
  • R16 - Tenor Check (TEIH): Due date minus issuance date must be less than TEIH days.
  • R17 - Issuance Age (ISSPUR): Purchase date minus issuance date must be at least ISSPUR days.

  1. Upload extraction data and select a cohort date.
  2. Apply rules to build candidates and record rule-based exclusions.
  3. Set limits in automatic (alpha/beta/gamma) or manual mode.
  4. Run the solver and review selected vs excluded results with reasons.
